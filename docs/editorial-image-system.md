# SCQR Editorial Image System

## Цель
SCQR больше не живёт на одном бумажно-оливковом визуальном шаблоне. Система должна удерживать узнаваемое лицо издания, но собирать его из нескольких редакционных языков:
- научный;
- человеческий;
- документальный;
- институциональный;
- предметный.

Текущая модель состоит из двух слоёв:
- `legacy`-слой для уже опубликованного архива;
- `primary`-слой для новых материалов, витрины и сильных редакционных зон.

## Главные правила
- Не каждая статья требует абстрактной метафоры.
- Пользовательское изображение всегда имеет приоритет над новой генерацией, если оно редакционно пригодно.
- Фирменный зелёный остаётся акцентом SCQR, но не должен доминировать в каждой обложке.
- На первой полосе должно одновременно жить минимум 3 разных визуальных режима.
- Соседние премиальные карточки не должны выглядеть как вариации одной и той же сцены.

## Слои системы

### Legacy / secondary
Используются для уже опубликованного архива и как fallback.
- `editorial_collage`
- `signal_network`
- `industrial_plate`
- `threshold_space`
- `quiet_monument`

### Primary / frontline
Используются для новых материалов, витрины, рубричных лидов и важных share-образов.
- `documentary_frame`
- `scientific_lucid`
- `human_collage`
- `civic_blueprint`
- `editorial_still_life`

## Primary-стили

### `documentary_frame`
- Живой редакционный кадр, человеческий масштаб, тёплые натуральные тона.
- Использовать для рабочих процессов, команд, городской инфраструктуры, рынков и реального внедрения.
- Избегать рекламной постановки и офисного стока.

### `scientific_lucid`
- Светлая научная сцена: лабораторная чистота, прозрачные материалы, ощущение исследования.
- Использовать для биологии, исследований, моделей, глубоких технологических релизов.
- Основная палитра: белый, серебро, мягкий кобальт, бордо, минеральный зелёный.

### `human_collage`
- Журнальный коллаж из фотофрагментов, жестов, документов, деталей лица, культурных следов.
- Использовать для языка, критики, культурных и социальных эффектов ИИ.
- Основная палитра: бумага, терракота, сливовый, графит, моховой.

### `civic_blueprint`
- Институциональная графика: карты, планы, архивы, государственные контуры, режимы допуска.
- Использовать для государства, регулирования, инфраструктурных процедур, архивной логики.
- Основная палитра: веллум, пыльно-синий, камень, слейт, civic green.

### `editorial_still_life`
- Предметная студийная сцена: один объект как метафора, мягкий свет, ясная композиция.
- Использовать для релизов, вычислительных архитектур, качества моделей, продуктовых сигналов.
- Основная палитра: лён, уголь, дымчатое стекло, оксидный красный, редкий зелёный акцент.

## Источники изображений
- `user-supplied`: картинка пришла от редакции/пользователя и проходит редакционный отбор.
- `generated`: картинка создана под статью с нуля.
- `hybrid`: пользовательский референс плюс генеративная переработка.
- `diagram`: схема или график как основной hero-образ.

Для `user-supplied` обязательно:
- осмысленный `heroAlt`;
- корректный `heroStyle`, описывающий реальный характер изображения;
- приоритетное использование в витрине, если изображение сильнее генеративной альтернативы.

## Выбор стиля
Стиль определяется не только рубрикой, но и сочетанием факторов:
- жанр статьи;
- тема и словарь материала;
- наличие пользовательской картинки;
- текущие стили на главной и в соседних карточках;
- роль материала: frontline / rubric lead / archive tail.

## Фронтовые правила разнообразия
- `homepageLead + homepageVisualGrid` должны давать минимум 3 визуальных режима.
- Не допускается серия из 3 похожих режимов подряд.
- В премиальные зоны не тянуть 4 карточки подряд из legacy-слоя без силькой причины.
- Хотя бы одна сильная зона первой полосы должна периодически опираться на `user-supplied` изображение.

## Контентные поля
Для новых материалов арт-контур описывается так:
- `heroStyle`
- `heroSource`
- `heroAlt`
- `heroImage`
- `deck`
- `scqrVerdict`

`heroStyle` — семейство визуальной режиссуры.  
`heroSource` — происхождение изображения и режим работы дизайнера.

## Файловая структура
- Основные пользовательские поставки: `src/assets/editorial/contributed/`
- Вариации и обновлённые серии: отдельные подкаталоги по дате и пакету
- Входящий запас на будущее: `src/assets/editorial/incoming/`

## Автоматизация
- Генератор брифов: `npm run image:briefs`
- Валидация контента: `npm run validate`
- Брифы собираются в [docs/editorial-image-briefs.md](/D:/CODEX/gitscqr/scqr/docs/editorial-image-briefs.md)

Если визуальный слой снова начинает скатываться в одну гамму и один тип абстракции, это не вкусовой вопрос, а сигнал, что дизайнерский контур работает недостаточно строго.
