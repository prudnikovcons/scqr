# SCQR Editorial Image System

## Цель
Держать все обложки SCQR в одном редакционном языке, но не в одном визуальном штампе. Система должна давать:
- узнаваемое лицо SCQR;
- 4–5 устойчивых стилевых направлений вместо одной повторяющейся манеры;
- абстрактную, метафорическую подачу без буквального “техно-стока”;
- рабочий контур, в котором для каждой статьи можно быстро собрать новый prompt-brief.

## Общие инварианты
- Формат: горизонтальная editorial cover image для карточки, статьи и share-preview.
- Характер: взрослый, аналитический, спокойный, без плакатной агрессии.
- База: светлая бумажная среда, мягкая фактура, ощущение журнального разворота.
- Акцент: кислотно-зелёный SCQR как единственный яркий цветовой сигнал.
- Композиция: один главный образ, максимум 1–2 вторичных сигнала, много воздуха.
- Запреты: никаких читаемых надписей, интерфейсов, логотипов, роботов, синих неонов, киберпанка, “AI stock image”.

## Пять стилевых направлений

### 1. Редакционный коллаж
- Внутренний id: `editorial_collage`
- Когда использовать:
  - колонки;
  - теории;
  - иллюзии;
  - сюжеты про язык, миф, безопасность, институциональные трения.
- Визуальный принцип:
  - бумажные слои;
  - вырезанные формы;
  - мягкий офсетный сдвиг;
  - интеллектуальный журнальный коллаж вместо “красивой CGI-картинки”.
- Подходит, когда статья строится не вокруг вещи, а вокруг смысла, конфликта интерпретаций или хрупкого баланса.

### 2. Сигнальная сеть
- Внутренний id: `signal_network`
- Когда использовать:
  - автоматизации;
  - протоколы;
  - агентные системы;
  - новые продуктовые запуски и инструментальные сюжеты.
- Визуальный принцип:
  - узлы, порты, каналы, связность;
  - модульная инженерная абстракция;
  - ощущение рабочей архитектуры, а не интерфейса.
- Подходит, когда важно показать, что рынок состоит из соединений, разрешений, потоков и уязвимых стыков.

### 3. Индустриальная пластина
- Внутренний id: `industrial_plate`
- Когда использовать:
  - инфраструктура;
  - вычисления;
  - энергия;
  - чипы;
  - тяжёлые рыночные сдвиги.
- Визуальный принцип:
  - металл, панели, щиты, стойки, силовые поверхности;
  - промышленная дисциплина;
  - материальность “нижних этажей” ИИ.
- Подходит, когда тезис статьи упирается в мощность, стройку, энергию, дефицит и реальную производственную базу.

### 4. Пороговое пространство
- Внутренний id: `threshold_space`
- Когда использовать:
  - генерации;
  - режимы доступа;
  - регулирование;
  - governance;
  - фронтирные лаборатории и сюжеты про допуск.
- Визуальный принцип:
  - шлюзы, рамки, коридоры, барьеры;
  - архитектурная логика доступа;
  - тихая институциональная власть.
- Подходит, когда материал о том, кто проходит дальше, кто не проходит, кто контролирует вход и где появляется новый режим допуска.

### 5. Тихий монумент
- Внутренний id: `quiet_monument`
- Когда использовать:
  - Россия;
  - рынок, капитал и власть;
  - крупные корпоративные ставки;
  - материалы, где нужен один большой символический объект.
- Визуальный принцип:
  - одна крупная масса;
  - много воздуха;
  - ощущение веса решения, капитала или институции.
- Подходит, когда сюжет не про сеть и не про схему, а про центр тяжести, власть, цену и масштаб.

## Мастер-промт системы
```text
Use case: stylized-concept
Asset type: editorial cover image for SCQR article
Primary request: create a premium editorial illustration for a Russian-language AI industry magazine
Scene/backdrop: calm paper-toned background with subtle print grain and soft studio depth
Subject: one clear metaphor that expresses the article thesis without literal UI screenshots or robots
Composition/framing: horizontal composition, strong focal point, generous negative space, one main object and one secondary signal, clean silhouette, built for website card and article hero
Lighting/mood: soft directional light, controlled contrast, serious and intelligent mood
Color palette: warm paper, charcoal, graphite, muted olive, SCQR acid green as a precise accent
Materials/textures: paper, brushed metal, smoked glass, matte plastic, faint ink and offset texture
Constraints: no text, no logos, no watermark, no people looking at camera, no generic stock-tech style, no readable interface, no blue neon cyberpunk, no clutter
```

## Как собирается частный prompt
В частный prompt всегда входят:
- `title`
- `deck`
- `scqrVerdict`
- `heroStyle`
- главный тезис статьи
- центральная метафора
- инструкция, где должен появиться зелёный акцент

В prompt нельзя опираться только на заголовок. Минимальная смысловая опора для генерации — это `deck + scqrVerdict`.

## Поля контента
Для каждой статьи теперь должны быть:
- `heroStyle`
- `heroAlt`
- `deck`
- `scqrVerdict`
- `heroImage`

`heroStyle` задаёт не конкретную картинку, а семейство визуальной режиссуры.

## Рабочий процесс
1. При создании статьи выставляется `heroStyle`.
2. Если это сильный материал для витрины, стиль при необходимости подтверждается вручную.
3. Скрипт `npm run image:briefs` собирает prompt-brief по архиву.
4. По brief генерируется новая обложка.
5. Для премиальной витрины используется premium-cover.
6. Для хвоста архива допускается `secondary`-обложка.

## Именование файлов
- Premium: `<slug>-hero-v2.png` или следующая явная версия при повторной генерации.
- Secondary: `secondary-<rubric-slug>-hero.svg`.
- Папка: `src/assets/editorial/`.

## Что избегать
- буквальный “робот с экраном”;
- неоновые футуристические города;
- синий корпоративный сток;
- интерфейсы, графики и текст внутри обложки;
- мемность, sci-fi-пафос и декоративный хаос;
- одинаковую композицию для всех материалов подряд.

## Практическое правило разнообразия
- Не ставить один и тот же стиль подряд на 3–4 соседние премиальные публикации.
- На главной стараться смешивать хотя бы 3 стилевых направления одновременно.
- Для рубрики допустима доминирующая манера, но не визуальная монокультура.

## Автоматизация

- Генератор prompt-brief: `scripts/export-image-briefs.mjs`
- Выгрузка brief-файла: `docs/editorial-image-briefs.md`
