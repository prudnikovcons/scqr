# Telegram-канал SCQR — пайплайн публикации

> Канал: [t.me/scqr_ai](https://t.me/scqr_ai). Цель — каждый материал на сайте автоматически появляется в канале в формате, удобном для мобильного чтения.

---

## Архитектурный принцип

Публикация в TG — отдельная стадия пайплайна, которая запускается **после** `git push` статьи в `main` и успешного Vercel-деплоя. На сайте — лонгрид с рубриками, тегами, версткой, обложкой. В TG — короткий хук с тезисом, ссылка обратно. Канал не дублирует сайт, он направляет на него.

Пайплайн стадии:

```
collect → pack → review → produce → publish → tg-post
                                              ↑ новое
```

## Состояние на 25 апреля 2026

- Канал создан владельцем.
- Сайт переведён на `https://ai.scqr.ru` — Open Graph и canonical корректны, поэтому TG-карточки превью отрисуются с заголовком, deck'ом и hero-картинкой автоматически по факту вставки ссылки.
- Все CTA на сайте указывают на канал (HomeNewsletterBanner, Header, Footer Telegram-icon, Footer "Помощь").
- Бот, токен, реальная отправка — пока не настроены. См. ниже что нужно от владельца.

## Что должен сделать владелец один раз

1. **Создать бота** через [@BotFather](https://t.me/BotFather):
   - `/newbot`
   - Имя: `SCQR Publisher` (или любое)
   - Username: `scqr_publisher_bot` (или любое доступное)
   - Получить токен формата `1234567890:ABCdef...`
2. **Добавить бота админом** в канал `@scqr_ai`:
   - Channel → Manage → Administrators → Add Administrator → найти бота
   - Права: только `Post Messages` (можно добавить `Edit Messages` для исправлений)
3. **Передать токен** Claude Code: положить в `.env.local` в корне workspace (`D:\CODEX\gitscqr\scqr\.env.local`):
   ```
   TELEGRAM_BOT_TOKEN=1234567890:ABCdef...
   TELEGRAM_CHANNEL=@scqr_ai
   ```
   Файл уже в `.gitignore`, в репо не попадёт.

После этого Claude Code может работать с каналом через Bot API.

## Что Claude Code умеет / не умеет

**Может:**
- Читать готовую статью из `site/src/content/posts/<slug>.md`
- Сформировать TG-сообщение по правилам из `docs/telegram-pipeline.md` (см. ниже)
- Отправить через Bot API (`POST https://api.telegram.org/bot<token>/sendMessage`)
- Зафиксировать `message_id` и время отправки в `decision_log`
- Поправить пост через `editMessageText` (если изменился исходник на сайте)
- Удалить пост (`deleteMessage`) — но только последние 48 часов и редко

**Не может (Telegram-ограничения, не Claude Code):**
- Создать бота — только @BotFather через telegram-аккаунт владельца.
- Назначить бота админом — только владелец канала.
- Опубликовать пост от своего имени без бота.

## TG-формат сообщения

Жёсткий шаблон. Каждый материал → один пост. Длина — 600–900 символов (с учётом разметки), чтобы умещалось на экран без свайпа на типичном мобильном.

```
{rubric_emoji} {type_emoji} <b>{title}</b>

{deck_or_first_paragraph_clipped_to_350_chars}

{rubric_label} · {type_label} · {readingTime} мин

<a href="{canonical_url}">Читать на SCQR →</a>

{hashtags}
```

### Эмодзи-коды

По рубрикам:

| Рубрика | Эмодзи |
|---|---|
| trajectories | 🧭 |
| generations | 🤖 |
| automations | ⚙️ |
| innovations | 💡 |
| illusions | 🌫️ |
| russia | 🧊 (нейтральная заметность) |
| regulations | 📜 |
| theories | 📚 |
| tendencies | 📈 |

По типу:

| Тип | Эмодзи |
|---|---|
| news | 📡 |
| analysis | 🔍 |
| column | 🖋️ |

### Хэштеги

Не более 4. Логика:
- 1 рубрика (русское написание без пробелов): `#тенденции`
- 1 тип, если он значим: `#аналитика`
- 1–2 ключевых topic из frontmatter, очищенных под TG: `#OpenAI` `#вычисления`

### Пример

Статья — `2026-04-24-criticgpt-kritik-dlya-modeley.md`, рубрика `generations`, тип `analysis`, тезис из deck, читается 2 мин.

```
🤖 🔍 <b>CriticGPT: критик для моделей</b>

CriticGPT интересен не как отдельный продукт для витрины, а как технологический цех. Чем лучше модель умеет критиковать другую модель, тем быстрее рынок строит фабрику качества вместо ручной разметки.

Генерации · Аналитика · 2 мин

<a href="https://ai.scqr.ru/2026-04-24-criticgpt-kritik-dlya-modeley/">Читать на SCQR →</a>

#генерации #аналитика #OpenAI #CriticGPT
```

При вставке ссылки TG автоматически подтянет hero, заголовок и description из Open Graph — большая карточка превью отображается под текстом.

### Когда отступать от шаблона

- **Колонки** (тип `column`) — длиннее, до 1200 символов: первый абзац целиком + якорь «Полная колонка →».
- **Серии (storyCluster)** — добавляется строка «Часть серии: <название>» перед deck'ом.
- **Дайджест** — отдельный шаблон (когда формат запустится): не одна статья, а 5-7 ссылок с короткими подписями.

## CLI-команды (после получения токена)

Запланировано в `engine/src/commands/tg.ts`:

```
pnpm scqr tg:preview <slug>          # собрать пост, показать в консоли, не отправлять
pnpm scqr tg:post <slug>             # собрать и отправить в @scqr_ai
pnpm scqr tg:edit <slug>             # обновить уже отправленный пост (по message_id из decision_log)
pnpm scqr tg:delete <slug>           # удалить (только в окне 48ч)
pnpm scqr tg:catchup                 # все материалы за последние N дней, которых нет в decision_log → опубликовать
```

Все вызовы пишут в `.scqr/logs/tg-<ts>.log` и в `decision_log` с `entity_type='article'`, `decision='tg_posted' | 'tg_edited' | 'tg_deleted'`, `metadata.message_id`, `metadata.chat_id`.

## Автоматизация после Vercel-деплоя

Цель: каждый успешный деплой `main` ветки → автоматически разнести новые статьи в канал.

**Вариант 1 (рекомендуемый): GitHub Action.**

`.github/workflows/tg-publish.yml`:
- триггер: `push` в `main`, paths `site/src/content/posts/*.md`
- шаги:
  1. checkout
  2. find новые/изменённые статьи через `git diff HEAD~1`
  3. для каждой — `pnpm scqr tg:post <slug>` (или `tg:edit` если уже была)
  4. секреты: `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHANNEL` в Settings → Secrets

**Вариант 2: Vercel Deploy Hook + Vercel Function.**

- Vercel webhook вызывает `https://ai.scqr.ru/api/tg-publish` после успешного билда
- Function читает sitemap.xml, диффит со списком уже опубликованных в канале, шлёт в TG
- Сложнее, требует API-роута на сайте.

**Вариант 3: ручной запуск.**

- Владелец/Claude после публикации статьи руками вызывает `pnpm scqr tg:post <slug>`
- Для теста — нормально. Для регулярки — нужен Variant 1.

Начнём с третьего, перейдём к первому, когда воркфлоу устаканится.

## Безопасность

- Bot token хранится только в `.env.local` (не коммитится) и в GitHub Secrets (для Action).
- `.env.local` уже в `.gitignore`.
- Если токен утечёт — отозвать через @BotFather (`/revoke`) и сгенерить новый. Канал не пострадает.
- Bot не имеет доступа к читателям канала — только постит.

## Чего избегать

- **Не дублировать body статьи в TG.** TG — указатель, а не зеркало. Сайт — каноническое место для лонгрида.
- **Не публиковать без Open Graph-проверки.** Если на сайте сломан OG (heroImage отсутствует, description пустой) — карточка превью будет пустой. Сначала чиним сайт, потом постим.
- **Не использовать MarkdownV2 для тела поста.** Слишком много экранирований. Используем `parse_mode=HTML` — устойчивее на длинных текстах с кириллицей и пунктуацией.
- **Не постить дайджест и единичные новости вперемешку.** В дайджест-выпуске — структурированный список со ссылками, в новости — карточка одной статьи. Разные шаблоны, разные кнопки.
