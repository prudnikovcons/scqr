# weekly-retro

## Когда срабатывает
- Пятница 16:00 (для будущего Codex-automation).
- Ручной запрос владельца.

## Вход
- `pnpm scqr retro <weekISO>` экспортирует JSON: decision_log + signals + reviews за неделю.
- `docs/memory/<previous-week>.md` — для контекста.

## Шаги
1. Получаешь экспорт (JSON или путь к файлу).
2. Спавнишь subagent `retro-reviewer` через Task tool.
3. retro-reviewer пишет `docs/memory/YYYY-WW.md` (4 раздела: что сработало / не сработало / паттерны / источники + подсказка следующему пакету).
4. Записывает в таблицу `memory` через CLI: `pnpm scqr memory:add <week> <topic> "<observation>"` (Phase 3+).

## Что попадает в memory
- Сюжетные линии, которые продлились (storyCluster ≥3 материалов).
- Источники, которые стабильно дают сигналы (или наоборот, шум).
- Тематические сдвиги в рецензиях владельца (новый угол, словарь).
- Решения о добавлении/заморозке источников.

## Жёсткие правила
- Не пишешь оценочные «хорошо/плохо» — только наблюдения с конкретной ссылкой на decision_log.
- Не повторяешь паттерн прошлой недели — обновляешь существующую memory-запись.
- «Подсказка следующему пакету» — 2-3 строки, не больше. Это вход для signal-collector.

## Выход
- `docs/memory/YYYY-WW.md`.
- Записи в таблицу `memory` (week, topic, observation, tags_csv, source: retro).
